vscode操作git

提示

本文出至我认识的一位大佬，前端12年开发经验，掘金地址:点击 (opens new window) 用vscode操作git确实要方便很多，因为有些配置性的文件我们提上去会有冲突，可能每个人电脑的环境所需要的配置不一样，写代码时拉一下代码，提代码时拉下代码，这样一般如果两个人各写各的模版，一般不会因为git操作而造成冲突，就算有冲突vscode上也能很好看谁和谁的代码冲突，就可以商量保留谁的代码

# git命令

# 常见命令

sh复制代码# 克隆仓库
#   以https协议
git clone https://xxx/xxx/xxx.git
#   以ssh协议
git git@xxx:xxx/xxx.git

# 取仓库别名
git remote add 别名 远程库路径
#   eg:git remote add origin https://xxx/xxx/xxx.git

# 查看仓库状态,红色文件代表未添加到'暂存区',绿色文件代表已添加到'暂存区'
git status

# 添加代码到本地仓库
#   添加单个文件
git add 路径/文件名
#   批量添加
git add .

# 提交代码到本地仓库
git commit -m '提交注释'

# 推送代码到远端
git push 别名/远程仓库路径 本地分支名
#   eg:git push origin master

# 拉取代码
git pull 别名/远程仓库路径 远程分支名
#   eg:git pull origin master

# 查看分支列表
git branch

# 创建分支
git branch 分支名

# 切换到分支
git checkout 分支名

# 创建并切换到分支
git branch 分支名 -b

# 合并分支
git merge 分支名
