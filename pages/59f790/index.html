# 1.style绑定变量

<script setup>
import {ref ,reactive} from 'vue'
 let color = ref('red')
 let size = ref(18)
</script>
<style>
style 中的绑定语法是：backgroundColor: v-bind(变量)
只能写表达式，比如高度为 100 + 200，必须写 v-bind(100+200+"px")
    .box{
        font-size:v-bind(size + 'px')
         color:v-bind(color)
    } 
</style>

# 2.vue3中的v-model

在 Vue3 中，.sync 被移除，一个组件可以有多个 v-model
使用方式是，props 传入 xxx，事件调用 emit('update:xxx')（与 vue2 .sync 保持一致）
父组件使用 v-model:xxx 绑定，当 xxx 为 modelValue 时，可以省略 :xxx 和 update:xxx

# 3.Dom节点的获取

1.第一种
<script setup>
import { ref, watch } from "vue";
const div = ref(null);
watch(div, (val) => {
  if (!val) return;
  val.style.color = "red";
});
</script>
<template>
  <div ref="div">content</div>
</template>
2.第二种
<script setup>
import { onMounted } from "vue";
let div = null; //没有响应式的
onMounted(() => {
  div.style.color = "blue";
});
</script>
<template>
  <div :ref="(el) => (div = el)">content</div>
</template>
当你希望视图跟随某个数据变化时，数据需要使用 reactive/ref 声明
但是，我们获取 dom 节点引用时，大部分情况下并不需要响应式，所以推荐使用 :ref="el => xxx=el" 的方式，在 onMounted 生命周期中获取 dom 引用
不过，这只是指导思想，需要根据具体场景选择获取 dom 节点的方式
判断标准是，是否要利用 watch 监听节点情况
