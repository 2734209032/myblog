# 1.下拉刷新

 //在json文件中配置
{
  "usingComponents": {},
  "enablePullDownRefresh": true  //开启支持	下拉
}
//页面相关事件处理函数--监听用户下拉动作
  onPullDownRefresh: async function () {
    this.setData({
      timestamp: Date.now(),
      list: [],         //清空数据，保证下拉刷新 是不同的内容
      timer: undefined
    })
    await this.getList()
    wx.stopPullDownRefresh()  //等到数据请求回来取消下拉刷新
  },
      
  getList(){
       return new Promise((resolve, reject) => {
      //防抖
      if (this.data.timer) {
        clearTimeout(this.data.timer)
      }
      // 提示加载中
      wx.showLoading({
        title: '加载中...',
      })
      let timer = setTimeout(async () => {
        // 方式3:通过第三方请求库
        let res = await wx.$http.get('/v1_0/articles', {
          timestamp: this.data.timestamp,
          channel_id: 0,
          with_top: 1
        })
        console.log(res)
        this.setData({
          timestamp: res.data.data.pre_timestamp,
          list: [...this.data.list, ...res.data.data.results]
        })
        resolve()//在这resolve 就能保证数据请求回来了，可以走await下面的代码了
        wx.hideLoading()//提示加载中取消
      }, 500)
      this.setData(timer)
    })
  },

# 5.组件

//创建组件
1.在项目的根目录中，鼠标右键，创建 components -> button-counter 文件夹
2.在新建的 components ->  button-counter 文件夹上，鼠标右键，点击“新建 Component ”
3.键入组件的名称button-counter之后回车，会自动生成组件对应的 4 个文件，后缀名分别为 .js ， .json ， .wxml和 .wxss
//注册组件
//全局注册 在app.json中
 "usingComponents": {
  "button-count": "/components/button-count/button-count"
  }
//局部注册 在index.json中
"usingComponents": {
    "local-btn-add": "../../components/button-count/button-count"
  }
//组件使用 在模版中
<button-count></button-count> ///全局
----------
<local-btn-add></local-btn-add> ///局部
//使用场景--------------------------------------------------------------
● 如果某组件在多个页面中经常被用到，建议进行“全局引用” 
●  如果某组件只在特定的页面中被用到，建议进行“局部引用” 
//组件和页面的区别---------------------------------------------------------
●  组件的 .json 文件中需要声明 "component": true 属性 
●  组件的 .js 文件中调用的是 Component() 函数 
  ○ 整个程序启动调用的是 App()
  ○ 某个页面的渲染调用的是 Page()
  ○ 某个组件的渲染调用的是 Component()
●  组件的事件处理函数需要定义到 methods 节点中

# 6.数据监听器

数据监听器用于监听和响应任何属性和数据字段的变化，从而执行特定的操作。 它的作用类似于 vue 中的 watch 侦听器
Component({
  data: {
    count: 0
  },
  methods: {
    onAdd() {
      console.log(this)
      this.setData({
        count: this.data.count + 1
      })
    }
  },
  observers: { //监听器
    count(val) { //只有一个参数，参数是新值
      console.log('cont', val)
    }
  }
})
//-----------------------------------注意-------------------------------------
1.只能在组件使用，普通页面是不能用的

# 7.组件的生命周期

//组件自身生命周期
1.created :相当于vue当中beforeCreated，数据初始化还没完成，不能调用this.setData方法和this.data.xxx
2.attached :相当于vue当中的created，数据初始化完成，可以调用this.setData方法和this.data.xxx，通常在这个钩子函数当中发送请求
3.ready ：相当于mouted
4.detached :相当于destoryed
//执行顺序
组件初始化 created -> attached -> ready
普通页面切换: detached (只能在普通页面实现，tabBar页面会一直存在，不会被销毁)
//定义的方式
1.可以直接在组件中的js定义 与methods同级 (不推荐)
2.在与methods同级的lifetimes对象中定义 (推荐 ，且优先级高)
Components({
    data:{},
    methods:{},
    lifetimes:{ 
    created() {
      console.log('created')
    },
    attached() {
      console.log('attached')
    },
    ready() {
      console.log('ready')
    },
    detached() {
      console.log('detached')
    }
    }
})

# 9.组件样式和页面样式的关系

1.默认情况下，自定义组件的样式只对当前组件生效，不会影响到组件之外的 UI 结构
2.app.wxss 中的全局样式对组件无效 ,页面的样式也不会影响组件的样式
//样式隔离的好处
① 防止外界的样式影响组件内部的样式
② 防止组件的样式破坏外界的样式
//注意点---------------------------------------------------------
● app.wxss 中的全局样式对组件无效 
●  只有 class 选择器会有样式隔离效果， id 选择器、属性选择器、标签选择器不受样式隔离的影响 
建议：在组件和引用组件的页面中建议使用 class 选择器，不要使用 id、属性、标签选择器！因为在绝 大多数情况下, 我们希望组件的样式是隔离开的

# 10.样式穿透

// 在组件的.js 文件中新增加如下配置
Component({
    options: {
         styleIsolation: 'isolated' // 默认值,代表启动样式隔离

{

# 11.父传子

1.在pages.js里定义数据
Page({
    data:{
        num:100
    }
})
2.在pages.wxml模版中绑定属性的方式传值
	<button-count num="{{num}}"></button-count>
3.子组件利用properties接收
Component({
    //简化写法
  properties: {
    num: Number，
    //完整写法
    num:{
      type:String, //类型
      value:'' //默认值
  }
  },}
         
  //子传父，提交事件
     1.子组件js中
     methods: {
    onAdd() {
      // console.log(this)
      // this.setData({
      //   count: this.data.count + 1
      // })
      this.triggerEvent('update-count', this.properties.num + 1)
    }
  },
    2.父组件的wxml模版中
    <button-count num="{{num}}" bind:update-count="onAdd"></button-count>
    3.父组件的js中
     onAdd(e) {
    console.log(e.detail)
     this.setData({
      num: e.detail
    })
  },

# 14.具名插槽

1.首先需要开启多个插槽的配置 组件的js中
Component({
    options: {
        multipleSolts: true // 在组件定义时的选项中启用多 slot支持
    },
    properties: { /* ...*/ },
    methods: { /* ...*/ }
})
2.定义
<view>
  <!-- name 为 header 的第一个 插槽 -->
  <slot name="header"></slot>
  <view>组件内部内容</view>
  <!-- 对于不确定的内容，可以使用<slot> 进行占位， 具体内容由组件的使用者决定 -->
  <slot></slot>
</view>
3.使用
<!-- 组件的使用者 -->
<my-pannel>
    <!-- 这部分内容将被放置在组件的 <slot>的位置上 -->
    <view slot="header">标题</view>
  	<view>通过插槽传到组件当中</view>
</my-pannel>

# 15.主包和分包

概念：
主包:打开小程序默认加载的页面(总的大小小于2M，不然就要分包),包含Tabar页面和pages页面
分包:打开小程序默认不加载的页面，只要路由跳转访问到页面的时候，才去微信服务器要页面(页面懒加载),需要单独配置

    {
      "root": "article_pkg", //分包的根路径
      "pages": [
        "pages/article/article" //分包的页面列表
      ]
