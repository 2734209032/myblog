# 1.v-if 和v-show指令

作用：控制元素的显示和隐藏

v-show

语法：
  语法   v-show="布尔值/布尔值的变量"
        本质上 ： 控制的是css的display:none
        使用场景：频繁的控制显示隐藏 、 做切换，就可以用v-show

v-if

语法：
  语法：v-if="布尔值/布尔值的变量"
        本质上：动态的控制节点的创建或者删除
        应用场景： 要么显示，要么隐藏，不那么频繁的切换的时候，就可以用v-if
        v-if 是惰性的， 如果v-if="false", 这个节点，不会创建

# 3.v-text 和v-html指令

底层是innertext和innerHtml

v-text

v-text="变量" 把变量解析普通文本插入到标签中 ,不能解析标签
        => 不常用，一般用的是插值语法{{  }}
1
2

v-html

 v-html 解析为html标签
            用法：有时候后端返回给我们一些带标签的字符，前端可以用v-html解析
            注意：不要把用户输入的内容，放到v-html里面, 可能会造成XSS攻击（跨站脚本攻击）
            v-html底层  ==> innerHTML 
            永远不要相信用户输入的内容！！！

# 5.vue就地复用策略

就地复用：vue会尽可能的同层级同位置， 对比虚拟dom，复用旧的dom结构，进行差异化更新
好处：可以复用旧的dom结构，提高渲染效率
设置了key，相当于给这个li标签加了一个标记，
新旧虚拟dom对比的时候，就会按着key的值 对比原来的那个标签
//就是相当于，当我们以数组的index作为key，然后比如我们在数组的开头加了一个元素，再去渲染页面时，首先根据标识一样的去对比，
//如果内容，一样就复用，否则生成，结构一样就复用，否则就生成，提高页面渲染的效率

# 7.diff算法

// diff算法  ==> 比较新旧虚拟dom的差异
// 策略1  ==> 同级根元素的对比
// 1. 如果根元素变化了，那么不考虑复用，整个dom树删除重建
// 2. 如果根元素不变，看根元素的属性是否变化来更新属性，并递归的往下对比差异。

// 策略2  ==> 兄弟元素比较
// 1. 对比同级兄弟元素，默认按照下标进行对比复用  （不加key或者说key为index的时候）
// 2. 如果设置了key，key是唯一的标识，会按照相同的key的元素进行复用

diff自己在网上找的

一.根据key对比新旧节点是否是相同类型的节点：
1. 不是相同节点：直接销毁旧的 vnode(虚拟dom)，渲染新的 vnode。这也解释了为什么 diff 是同层对比
1.1如果新 vnode 是文字， vnode就直接调用浏览器的 dom api 把节点的直接替换掉文字内容就好。
1.2 如果新 vnode 不是文字， vnode如果有新 children 而没有旧children说明是新增 children，直接 addVnodes 添加新子节点。有旧 children 而没有新 children说明是删除 children，直接 removeVnodes 删除旧子节点如果新旧 children 都存在（都存在 li 子节点列表，进入👈）
二.如果新旧虚拟dom中的key没有一个对应，哪怕新旧虚拟dom结构内容一致，也会把旧虚拟dom全部删除，直接把新的虚拟dom创建节点
