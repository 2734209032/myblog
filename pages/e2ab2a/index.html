# 1.你了解vue3吗

1.Vue3 于 2020 年 9 月发布
2.它性能更好，体积更小，支持 Typescript，并保持了向后兼容
3.它带来了 Composition API、依赖注入、Teleport、Suspense、v-model 等新特性
4.一般采用 Vite 开发构建 Vue3 项目

# 2.你了解vite吗

1.Vite 是 Vue3 推荐的构建工具，但不止用于 Vue3
2.Vite 冷启动速度快，热更新也很快，并且配置简单
3.Vite 在开发环境下浏览器原生 esm，因此开发体验更好4.但是，Vite 依然存在白屏问题，生产环境性能提升有限，无法使用 cjs 库等问题

# 3.vue中如何阻止重复请求

1.//通过定时器做防抖处理
效果:当用户连续点击多次同一个按钮,最后一次点击之后,过小段时间后才发起一次请求
原理:每次调用方法后都产生一个定时器,定时器结束以后再发请求,如果重复调用方法,就取消当前的定时器,创建新的定时器,等结束后再发请求,工作当中可以用第三方封装的工具函数例如lodash的debounce方法来简化防抖的代码
 methods: {
        // 调用lodash的防抖方法debounce,实现连续点击按钮多次,0.3秒后调用1次接口
        onClick: _.debounce(async function(){
            let res = await sendPost({username:'zs', age: 20})
            console.log('请求的结果', res.data)
        }, 300),
    },
})
2. //通过axios请求拦截器取消重复请求
通过axios请求拦截器,在每次请求前把请求信息和请求的取消方法放到一个map对象当中,并且判断map对象当中是否已经存在该请求信息的请求,如果存在取消上传请求

# 5.vue3的新特性

1.setup 是 Vue3.x 新增的一个选项， 他是组件内使用 Composition API的入口。它是最先执行的，位于onbeforCreacte之前
2.组合式api的写法，任何api都需要引入再使用
3.之前我们vue2的响应式变量，需要写在data中，vue3中响应式变量，需要用ref 或reactive声明
4.Teleport传送门 是一种能够将我们的组件的html结构移动到指定位置的技术，Suspense组件 等待异步组件时渲染一些额外的内容，让应用有更好的用户体验
5.vue2中只能有一个根节点，虽然vue3中写法上可以不需要根节点，但其实是用Fragment已经设置好了根节点

# 7.vue中的route-link和HTML中的a有什么区别

1.Router-link会转化成a标签，但是里面的path属性会转化成路由地址 + 锚链接 ，另外router-link是 vue-router提供的组件
2.HTML 中的 <a> 标签是常规的超链接元素，它可以用于跳转到普通的 HTML 页面、下载文件或者发送邮箱等。与信使网页不同,使用<a>标签进行页面内路由切换，需要刷新整个页面才能显示新的内容,用户体验上欠缺了许多.

# 8.vue中，为什么computed定义的方法不能带参数?而methods定义的方法又可以带

1.计算属性 有缓存功能，他定义为属性，也就是说 他没有小括号的调用方式，是框架内部实现了 该函数的调用 没有传参数的空间
2.而方法，相当于 一个函数，可以在标签上 使用 调用方式来传值 比如 @click="test(123)"
